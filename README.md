## 📖 О проекте

Это лендинг под казино, созданный командой **LNKX**. Приложение представляет собой Full-Stack решение для арбитража трафика и перенаправления пользователей на офферы (мини-игры):
- **Frontend**: Написан на React + TypeScript (Vite) с использованием TailwindCSS.
- **Backend (Edge)**: Легковесный сервер на базе Hono, работающий на Cloudflare Workers. 
- **База данных**: Используется Cloudflare KV для сверхбыстрого хранения статистики переходов и аналитики.

Проект собирает информацию о кликах (IP, страна, устройство, браузер) и предоставляет защищенную паролем админ-панель (`/panel`) для просмотра этих данных.

## 🎮 Как добавлять мини-игры и менять картинки

Список игр и их картинки хранятся прямо в коде React-фронтенда. Чтобы изменить обложку или добавить новую игру, найдите массив `GAMES` в компоненте списка игр (например, в `src/App.tsx` или похожем файле).

Он выглядит так:
```javascript
const GAMES = [
  {
    id: "doodleJump",
    title: "Doodle Jump",
    img: "https://cdn.beacons.ai/user_content/.../doodleJump.webp",
  },
  {
    id: "mineslot",
    title: "Mineslot",
    img: "https://cdn.beacons.ai/user_content/.../mineslot.webp",
  },
  // ... и так далее
];
```

**Памятка по изменению:**
1. **Замена картинки:** просто поменяйте ссылку в поле `img` на нужную вам.
2. **Добавление игры:** добавьте новый блок `{ id: "...", title: "...", img: "..." }` в этот массив.
3. **Ссылки для игр:** после добавления новой игры, не забудьте прописать её ссылку-редирект (реф-ссылку на казино) в переменные окружения `wrangler.toml` и добавить в `server.ts` (по аналогии с `DOODLE_JUMP_LINK`).
4. **ОБЯЗАТЕЛЬНО:** После любых изменений в файлах `.tsx`/`.ts` фронтенда, обязательно запускайте сборку (`npm run build`), иначе сервер будет отдавать старую версию дизайна!

## 🔐 Админ-панель и Секретные переменные (Environment Variables)

Для защиты статистики и настройки ссылок используются переменные окружения (Secrets/Vars). 
Все они прописываются в файле `wrangler.toml` (раздел `[vars]`) или задаются напрямую в панели Cloudflare.

**Список переменных:**
- `ADMIN_PASSWORD` — пароль для доступа к админ-панели (по умолчанию `overload`).
- `TELEGRAM_LINK` — ссылка на Telegram канал/бота.
- `GAME_LINK`, `DOODLE_JUMP_LINK`, `MINESLOT_LINK`, `AVIAMASTERS_LINK`, `TOWER_RUSH_LINK` — ссылки на соответствующие мини-игры/казино-офферы.

**Вход в Админ-панель:**
Админка доступна по адресу `ваш-домен.com/panel`. 
Введите пароль, указанный в переменной `ADMIN_PASSWORD`.

## ☁️ Деплой на Cloudflare (Памятка по загрузке в интернет)

Проект спроектирован для мгновенного деплоя в Cloudflare (Workers). Эти команды помогут вам легко загрузить или обновить сайт на сервере:

1. **Авторизация (делается один раз)**:
   ```bash
   npx wrangler login
   ```
   *Откроется браузер, где нужно войти в аккаунт Cloudflare.*

2. **Загрузка кода в продакшен**:
   ```bash
   npm run deploy
   ```
   *Эта удобная команда автоматически сделает `npm run build` (чтобы обновить изменения фронтенда и картинок игр), а затем выгрузит свежий код на Cloudflare.*

## 🚀 Как запустить на своём ПК (Локально для тестов)

1. Установите библиотеки:
   ```bash
   npm install
   ```
2. Соберите фронтенд в статику (папку `dist`):
   ```bash
   npm run build
   ```
3. Запустите локальный сервер:
   ```bash
   npm run dev
   ```
*Сайт будет доступен по адресу `http://localhost:8787`.*

## 🌍 Деплой на обычный хостинг (VPS / Nginx)

Если вы решите разместить проект на обычном хостинге, учтите архитектуру:
- **Фронтенд**: можно выполнить `npm run build` и всю папку `dist` закинуть на любой веб-сервер (Nginx, Apache).
- **Бэкенд**: часть со сбором статистики и админкой (`server.ts`) жестко привязана к Cloudflare KV. Чтобы она заработала на обычном сервере, потребуется переписать её под Node.js + базу данных SQLite.
